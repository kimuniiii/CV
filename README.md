# 自己紹介

私は大学時代は理系でしたので、大学時代からプログラミングを学んでいました。

社会人になってから本格的に`Web系のフロントエンジニア`としてキャリアを歩み、  
Sierでの受託開発、自社開発企業での自社サービス開発のどちらも経験して参りました。

既存機能の追加・改修と新規開発のどちらも経験しているため、  
プロジェクトに応じてどちらでも柔軟に対応することが可能です。

技術的な得意分野は`WEBフロントエンド領域における新規開発`です。

`React・TypeScript・Next.js`を用いた新規開発をメインにこなしてきたため、WEBフロントエンド領域の業務範囲は一通り網羅しています。

メンバーに実績を認めていただき、自分が主催となって社内勉強会の登壇を行った経験もございます。

また直近のプロジェクトでは、実績を認めていただき、新規開発PRJのフロントエンドの開発リーダーに抜擢していただきました。

業務における強みとしては、仕様を決める・詰める・確認する能力があることです。

これまで多くの仕様が不透明なプロジェクトに携わってきましたが  
仕様が不明瞭なままでは実装を進めることはできない、かつ、手戻りが発生する確率が格段に高くなります。

ですので、手を動かすよりも先にまずは仕様を詰める・仕様をまとめることを優先して動いてきました。

結果、社内の上位層の方から仕様を詰める能力が高く評価され、月当たりの単価が上がった経験もあります。

今後も今までの経験分野だけに限定することなくその案件で必要とされている技術やビジネス知識を身に付けていき  
プロジェクトを俯瞰視して仕事ができる「上流工程から逆算して考えられるエンジニア」になることを目指しています。

将来的には、新規開発のテックリードとしてプロジェクトを完遂・成功させることができるエンジニアになっていく予定です。
